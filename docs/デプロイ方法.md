# デプロイ方法

このBotは GitHub Container Registry (GHCR) に Docker イメージとして公開され、Watchtower によって自宅サーバー上で自動更新されます。

## 仕組み

```
GitHub Actions                    自宅サーバー
─────────────                    ──────────────────────────────────────
release.yml が実行               Watchtower（5分ごとにポーリング）
  ↓                                ↓
Dockerイメージをビルド   →   新しいイメージを検知 → bot コンテナを自動更新
  ↓
GHCRにpush (ghcr.io/aktnb/discord-bot-go:latest)
  ↓
GitHub Releaseを作成
```

- **SSH不要**：サーバー側からGHCRをポーリングするため、自宅のIPを公開する必要はありません
- **マイグレーション自動実行**：`bot` コンテナ起動時に `migrate up` が自動で実行されます

## 初回セットアップ

### 1. 専用ユーザーの作成

rootで実行します。

```bash
# botユーザーを作成（ログインシェルなし）
sudo useradd -r -m -s /bin/bash discord-bot

# botユーザーに切り替え
sudo -i -u discord-bot
```

### 2. Docker のインストール

```bash
# 既存パッケージの更新
sudo apt-get update
sudo apt-get install -y ca-certificates curl

# Docker の公式GPGキーを追加
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/debian/gpg \
  -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

# Docker リポジトリを追加
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] \
  https://download.docker.com/linux/debian \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Docker をインストール
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
```

> [!NOTE]
> Ubuntu の場合は `https://download.docker.com/linux/ubuntu` を使用し、
> `/etc/apt/keyrings/docker.asc` のURLも `ubuntu` に読み替えてください。

### 3. discord-bot ユーザーを docker グループに追加

rootで実行します。

```bash
sudo usermod -aG docker discord-bot

# グループ変更を反映（再ログインが必要）
sudo -i -u discord-bot
```

Docker が使えることを確認します：

```bash
docker info
```

### 4. 作業ディレクトリの作成

```bash
mkdir -p /home/discord-bot/app
cd /home/discord-bot/app
```

### 5. docker-compose.deploy.yml の配置

```bash
curl -fsSL \
  https://raw.githubusercontent.com/aktnb/discord-bot-go/master/docker-compose.deploy.yml \
  -o docker-compose.deploy.yml
```

### 6. 環境変数ファイルの作成

```bash
cat > .env <<'EOF'
DISCORD_TOKEN=your_discord_bot_token_here
DATABASE_URL=postgres://botuser:botpass@db:5432/botdb?sslmode=disable
POSTGRES_USER=botuser
POSTGRES_PASSWORD=botpass
POSTGRES_DB=botdb
EOF
```

`DISCORD_TOKEN` と `POSTGRES_PASSWORD` を実際の値に書き換えます：

```bash
nano .env
```

> [!CAUTION]
> `.env` ファイルは機密情報を含みます。パーミッションを制限してください。
>
> ```bash
> chmod 600 .env
> ```

### 7. GHCRからイメージをpull

```bash
docker compose -f docker-compose.deploy.yml pull
```

### 8. 起動

```bash
docker compose -f docker-compose.deploy.yml up -d
```

起動すると以下のコンテナが立ち上がります：

| コンテナ | 役割 |
|---|---|
| `db` | PostgreSQL データベース |
| `bot` | Discord Bot（起動時にマイグレーション実行後、Botを起動） |
| `watchtower` | 5分ごとにGHCRをチェックし、新しいイメージがあれば自動更新 |
| `portainer` | Dockerコンテナの管理UI（`http://サーバーIP:9000`） |

### 9. 動作確認

```bash
# 全コンテナの状態を確認
docker compose -f docker-compose.deploy.yml ps

# Botのログを確認
docker compose -f docker-compose.deploy.yml logs -f bot
```

`bot` ログに以下が出ていれば正常起動しています：

```
migrate up: no change  (or applied N migrations)
Bot is running. Press CTRL-C to quit.
```

### 10. systemd でサーバー再起動時に自動起動（任意）

サーバー再起動後もコンテナが自動で立ち上がるよう systemd サービスを設定します。

```bash
sudo tee /etc/systemd/system/discord-bot.service <<'EOF'
[Unit]
Description=Discord Bot (docker compose)
After=docker.service
Requires=docker.service

[Service]
Type=oneshot
RemainAfterExit=yes
User=discord-bot
WorkingDirectory=/home/discord-bot/app
ExecStart=docker compose -f docker-compose.deploy.yml up -d
ExecStop=docker compose -f docker-compose.deploy.yml down
TimeoutStartSec=120

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable discord-bot
sudo systemctl start discord-bot
```

## 以降のデプロイ

コードをmasterにマージして `release.yml` ワークフローを実行すると、GHCRに新しいイメージがpushされます。Watchtowerが最大5分以内に検知し、`bot` コンテナを自動更新します（マイグレーションも自動実行）。

手動で即時更新したい場合：

```bash
cd /home/discord-bot/app
docker compose -f docker-compose.deploy.yml pull
docker compose -f docker-compose.deploy.yml up -d
```

## トラブルシューティング

### Botが起動しない

```bash
docker compose -f docker-compose.deploy.yml logs bot
```

マイグレーションエラーの場合は `DATABASE_URL` の設定を確認してください。

### Watchtowerが更新しない

```bash
docker compose -f docker-compose.deploy.yml logs watchtower
```

GHCRのイメージが `linux/arm64` でビルドされているか確認してください（`release.yml` の `platforms: linux/arm64`）。

### docker コマンドが permission denied になる

`discord-bot` ユーザーが `docker` グループに属しているか確認します：

```bash
groups discord-bot
```

`docker` が含まれていない場合は手順3を再実行し、再ログインしてください。
